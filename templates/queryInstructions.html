How to specify the data type in a query

The Might Mongo uses a regular-expression based syntax for specifying the data type for a query. This is important, for instance, to differentiate between the string "42" and the number, 42. The formatting is based generally on Javascript syntax. As you type, the field will automatically color-code to indicate that you've matched a particular data type. A sample box is available at the bottom so you can practice!

  • For a boolean, just type true or false
  • Any string of digits from 0 to 9 is a number. An optional decimial point or negative sign works too. Examples: 42, 42.2, -5
  • Anything inside of double quotes ("...") is a string. For example, "Hello world", "42"
  • A date/time has two formats. One with just a date and one with a date & time. Both start with "dt:". Just a date has the format dt:YYYY:MM:DD. For example: dt:2014-12-10. For Date and time, use the format dt:YYYY-MM-DDTHH:MM:SS. For example: dt:2014-12-10T12:12:12
  • For a nullType (some languages called it 'none'), just type null
  • For an objectID, type oid:oid_code where oid_code must be a 24-character hexadecimal (0-9 and a-f) string of characters. i.e. oid:4f13b4296118ad10e7050602
